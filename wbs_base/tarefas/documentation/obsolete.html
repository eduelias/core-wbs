Tasks

Why aren't deleted tasks removed from the database?

When you click the 'Delete' button in tasks, the task is not actually removed from the database - instead it is flagged as having been deleted. There are several reasons why I decided to leave deleted tasks in the database instead of actually removing them. Here are the big ones.

  1. Oops - People have a tendency to delete things then say 'oops' and want to get them back again. If tasks are actaully removed from the database you can't get them back without some serious heroic measures.
  2. Spiders/Web Crawlers - This is software designed to run on the web. Not everyone password protects their tasks installation and spiders/web crawlers could come through and delete all of their tasks. I've had this happen on the demo before.
  3. Why not? - Database speed isn't noticably affected by having a couple hundred or thousand extra rows in it. If you have a million+ records let me know if it makes a difference for you.
  4. What do we gain? - I already exclude deleted tasks from the search results and I have introduced the option of hiding deleted tasks in the history list, so if you don't want to see them - you won't (unless you key in the ID of a deleted task into the URL directly).

If you think you have a good argument for changing the way this works - feel free to let me know.
